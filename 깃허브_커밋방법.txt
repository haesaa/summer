! 처음 쓰는데 누군가 이전에 이 기기를 사용해서 깃허브에 접속을 했던 이력이 있는 경우 초기화 하는 깃 명령어

git config --global --unset user.name
git config --global --unset user.email

! 기변을 했거나 깃허브를 아예 처음 쓰는 경우 사용

git config --global user.name "내 이름"
git config --global user.email "이메일 주소"

git init - 초기화

git status - 상태 확인

git add .gitignore - gitignore 파일 작성 (api 키를 쓸 때 환경 변수를 만든 경우, DB를 쓰고 있는 경우, 그 외 보안과 밀접한 관련이 있는 요소들을 추가, 작성은 vscode, notepad 등과 같은 편집기 프로그램에서 작성 가능함)

git commit -m "feat: Add .gitignore for Python/Django project" (gitignore 커밋을 해야 제대로 올라감)

git add . (전체 파일 추가, 이 경우 gitignore를 먼저 작성을 했기 때문에 그곳에 작성이 된 것들은 무시됨)

git status (gitignore에 올린 것이 추적이 되고 있는 지 여부를 보기 위해 사용)

! 계속 추적이 되고 있는 경우에 사용하는 명령어

git rm-cache 파일 경로 (gitingore에 작성을 한 실제 파일이 있는 경로, 탐색기를 열어서 직접 보는 것을 권장)

git commit -m "first commit" - 최종 커밋을 위한 메세지 작성

git branch -M main 작업 공간 생성

git remote add origin "본인의 깃허브 레포지토리 주소" (origin - 작업 공간 이름)

! 작업 공간이 이미 존재해서 올리지 못하는 경우 (디렉터리를 바꾼 상태로 같은 레포지토리에 올리는 경우가 제일 대표적임)

git remote -v - 원격 저장소 확인
gir remote rm 원격 저장소 이름 (현재 원격 저장소 이름은 origin이므로 origin을 쓰면 되고, 본인이 어떻게 지정을 하느냐에 따라 달라짐)

git push -u origin 작업 공간 이름 (branch에 적은 이름) - 최종 커밋 완료를 위한 명령어 (보통 특별한 경우가 없다면 main으로 명명)

git commit -m "커밋 메세지"

! 새로운 작업 공간을 만들 경우

git branch -M 작업공간 이름 (보통 잘 만들지 않음)

git push -u origin 작업 공간 이름 (main으로 보통 설정하므로 main을 적으며, 이미 기존 커밋 기록이 있어서 알아서 올라감)
